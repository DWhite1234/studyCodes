# this关键字

```java
1.代指当前对象
2.可以用来区分成员变量和局部变量
3.可以调用构造器和方法
4.在构造器中调用构造器时,必须放在首行

注意点:不能和super同时使用
```

# super关键字

```java
1.代指从父类继承来的资源
2.可以调用父类构造器和方法
3.在子类构造器中调用构造器时,必须放在首行


注意点:不能和this同时使用
```

# 多态

```java
1.向上转型
Animal a=new Cat();
用于同一个同时返回多个子类类型,减少代码量

例如:
public Animal show(Animal a){
    if(...){
        return new Cat();
    }
    if(...){
        return new Dog();
    }
    return null;
}

注意点:使用向上转型后就不能使用子类的独有方法
如果需要使用子类独有方法,应该使用向下转型
2.向下转型
例如:
	Animal a=new Cat();
    Cat cat=(Cat)a;

```

# native

```java
native: 修饰方法  本地方法
        public native int hashCode();
注意：
    0.java不是万能的有些功能无法实现  需要借助于 C/C++ 语言实现
    1.native方法没有方法体
    2.本地方法也是可以重写的
    3.本地方法执行时 在 本地方法栈中开辟内存
```

# final

```java
1.修饰类  类不可继承
2.修饰成员变量 成员变量成常量,不可变
3.修饰方法 方法不可以被重写
4.修饰对象：地址值不能改变,属性值可以改变
```

# Object类Api

```java
1.toString
2.getClass,获取运行时类型
3.finalize,当对象被GC确定为要被回收的垃圾，在回收之前由GC帮你调用这个方法
4.hashCode:
	hashcode不同,对象一定不相同
	hashcode相同,对象不一定相同
5.equals:用于比较两个对象是否相等,该方法没有重写前和==相同,比较地址值
	如果重写equals,则必须要重写hashcode
```

# 抽象类

```java
抽象类的语法格式:
【权限修饰符】 abstract class 类名{
    
}
【权限修饰符】 abstract class 类名 extends 父类{}

抽象方法的语法格式:
【其他修饰符】 abstract 返回值类型  方法名(【形参列表】);

注意：抽象方法没有方法体

1.抽象类可以没有抽象方法,但是有抽象方法的一定是抽象类
2.abstract修饰方法,方法没有方法体,子类必须重写父类的所有的抽象方法,或者将自己变成抽象类
```

# 虚方法和非虚方法

```java
非虚方法：不能重写的方法
虚方法：可以重写的方法
        静态分派：看这个对象的编译时类型，在这个对象的编译时类型中找到最匹配的方法
        动态绑定：看这个对象的运行时类型，如果这个对象的运行时类重写了刚刚找到的那个最匹配的方法，
        那么执行重写的，否则仍然执行刚才编译时类型中的那个方法
```
